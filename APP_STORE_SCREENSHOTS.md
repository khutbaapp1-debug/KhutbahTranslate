# Khutbah Companion - Screenshot Strategy Guide

Complete guide for creating app store screenshots for iOS and Google Play across all markets.

---

## 📐 TECHNICAL REQUIREMENTS

### iOS App Store (2024-2025 Updated)

**MANDATORY iPhone Screenshots:**
- **6.9" Display (iPhone 16 Pro Max)** - REQUIRED
  - Portrait: 1320 × 2868 pixels
  - Landscape: 2868 × 1320 pixels
  - Apple auto-scales to smaller devices from this size

**Optional (recommended for better compatibility):**
- 5.5" Display: 1242 × 2208 pixels (legacy support)

**iPad Screenshots (if supporting iPad):**
- **13" Display (iPad Pro 13-inch)** - REQUIRED
  - Portrait: 2064 × 2752 pixels
  - Landscape: 2752 × 2064 pixels

**File Specifications:**
- Format: JPEG or PNG
- No transparency
- No layers
- Resolution: 72 PPI
- Color space: RGB
- Quantity: Minimum 1, Maximum 10
- File size: 8 MB max per image

---

### Google Play Store

**Phone/Tablet Screenshots:**
- Aspect ratio: 16:9 (landscape) or 9:16 (portrait)
- Recommended: 1080 × 1920 pixels (portrait) or 1920 × 1080 pixels (landscape)
- Minimum dimension: 320 pixels
- Maximum dimension: 3840 pixels
- Format: JPEG or 24-bit PNG (no alpha)
- File size: 8 MB max per screenshot
- Quantity: Minimum 2, Maximum 8 (recommend at least 4)

**Feature Graphic (Google Play ONLY):**
- Size: 1024 × 500 pixels
- Format: JPEG or 24-bit PNG
- **REQUIRED** for Google Play
- Shows in store listing header

## 🎨 SCREENSHOT CONTENT STRATEGY

### Rule of First 3
**The first 3 screenshots are the most important** - most users never scroll past them.

### Screenshot Sequence (Priority Order)

#### Screenshot 1: HERO - Live Translation in Action
**What to show:**
- Split screen: Arabic khutbah text on top, English translation below
- Real-time translation happening (show translation appearing word-by-word or scrolling)
- Mosque/Islamic background blur
- Clear, large text showing both languages

## 📱 SCREENSHOT TEXT OVERLAY GUIDELINES

### DO:
✅ Use large, readable fonts (minimum 32pt)
✅ High contrast text (white on dark, dark on light)
✅ Keep text to 1-2 short lines maximum
✅ Use benefit-driven copy (not just feature names)
✅ Include visual hierarchy (headline + subheading)
✅ Use emojis sparingly for visual interest (🕌 🤲 📖)

### DON'T:
❌ Use promotional phrases like "Best App Ever"
❌ Make screenshots text-heavy (≤20% text coverage)
❌ Use low-contrast colors
❌ Include fake phone frames (full bleed only)
❌ Show people's faces without releases
❌ Include competitor names or branding

## 🔧 PRODUCTION TOOLS & TIPS

### Recommended Tools
1. **Figma** - Design screenshots with templates
2. **AppLaunchpad** - App store screenshot templates
3. **Previewed** - Device mockup generator
4. **Xcode Simulator** - Capture actual app at exact resolution
5. **Fastlane Snapshot** - Automate screenshot generation

### Capturing from Actual App
```bash
# For iPhone 6.9" (iPhone 16 Pro Max simulator in Xcode)
1. Open Xcode
2. Select iPhone 16 Pro Max simulator
3. Run app and navigate to feature
4. Cmd + S to save screenshot
5. Screenshots saved to Desktop by default
```

### Design Flow
1. **Design in Figma** at exact resolution (1320 × 2868 for iPhone)
2. **Export as PNG** at 72 PPI
3. **Add text overlays** in separate layer
4. **Test visibility** at smaller preview sizes
5. **A/B test** different variations using store tools

---

## 📊 A/B TESTING STRATEGY

### Apple Product Page Optimization
- Test up to 3 screenshot variations
- Run for minimum 2 weeks
- Track conversion rate improvements
- Test one element at a time (e.g., screenshot 1 text)

### Google Play Store Listing Experiments
- Test multiple screenshot orders
- Run for minimum 7 days
- Compare tap-through rates
- Test different text overlays

### What to Test
1. **Screenshot 1 variations:**
   - Different hero messages
   - Translation visualization styles
   - Text vs no text overlay

2. **Screenshot order:**
   - Features first vs translation first
   - Premium placement (position 4 vs 7)

3. **Text overlay styles:**
   - Short vs long copy
   - Benefit-led vs feature-led
   - Emoji usage vs clean text

## 💡 PRO TIPS

### Storytelling Flow
Think of screenshots as a visual story:
1. **Problem** (Screenshot 1): "Can't understand Arabic khutbah"
2. **Solution** (Screenshot 2-3): "Real-time translation + easy access"
3. **Features** (Screenshot 4-6): "Plus all these Islamic tools"
4. **Upgrade** (Screenshot 7): "Get even more with Premium"
5. **Trust** (Screenshot 8): "Join our community"

### Visual Consistency
- Use same phone mockup style throughout (or full bleed)
- Consistent color scheme (teal, gold, white, black)
- Same text overlay style and positioning
- Unified iconography and design language

### Optimization Mindset
- Monitor conversion rates weekly
- Update screenshots quarterly (or when adding major features)
- Test during high-traffic periods (Ramadan, Fridays)
- Respond to user feedback about confusion/expectations
