Khutbah Translate (Full Build Instruction)

🎯 PROJECT TITLE:
Build a Full-Stack Real-Time Arabic-to-English “Khutbah Translate” Web Application

⸻

🔧 TASK

Build a complete full-stack web application called Khutbah Translate, which performs real-time translation of Arabic khutbahs into English while respecting Islamic terminology, cultural tone, and religious etiquette.

The app should:
	1.	Record live Arabic audio continuously during a khutbah (no pauses).
	2.	Transcribe speech to Arabic text in real time using OpenAI Whisper (streaming or chunk-based).
	3.	Translate each transcript chunk into English in real time, using GPT/Gemini via Lovable AI Gateway.
	4.	Preserve Islamic terminology, Qur’anic references, and cultural meaning.
	5.	Automatically append “peace be upon him” whenever Prophet Muhammad ﷺ is mentioned.
	6.	Identify Qur’anic verses and mark them respectfully.
	7.	Provide real-time translated subtitles with smooth, continuous updates.

⸻

🌙 RELIGIOUS RESPECT REQUIREMENTS

Replit must implement custom logic:
	•	Always preserve Islamic terms (e.g., salah, taqwa, jannah, zakah, ihsan).
	•	Never mistranslate Allah, Rasulullah, SWT, Jannah, Jahannam, etc.
	•	Add “(peace be upon him)” automatically after any mention of Prophet Muhammad, or “Rasulullah,” or “the Messenger.”
	•	Detect Qur’anic recitation and format it as a Qur’an block with markers.
	•	Maintain scholarly tone and cultural sensitivity throughout translations.

🔄 SYSTEM ARCHITECTURE

Recording Flow
	1.	User clicks Start Recording
	2.	Microphone data captured in small WebM chunks
	3.	Base64 audio → transcribe-audio edge function
	4.	Whisper converts speech → Arabic text
	5.	Arabic text sent to translate-text function
	6.	English translation returned instantly
	7.	UI updates a live “translation feed”

Data Flow

Client → Edge Function → OpenAI/Lovable → Supabase → Back to Client
	•	Users can create, read, update, delete their own sermons only
	•	RLS ensures full privacy
